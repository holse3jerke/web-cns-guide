# web-cns-guide

本仓库用于归档和发布若干独立 HTML 页面。这些页面内容各自独立，不针对任何特定域名或网站，旨在提供一种结构化的信息呈现方式。

## 项目简介

本仓库主要服务于以下目的：
- 归档多个自包含的 HTML 文件
- 通过 GitHub Pages 或类似服务发布静态页面
- 提供清晰的目录结构，便于浏览和引用

所有页面均为独立 HTML 文件，不依赖外部数据库或后端服务。页面内容可能涉及指南、说明或参考信息，具体见各文件内部描述。

## 目录说明

```
/
├── index.html         # 仓库入口页，通常包含导航或总览
├── pages/             # 存放各独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/            # 公共资源目录（可选）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md          # 本文件
```

- `index.html`：仓库首页，建议作为浏览起点。
- `pages/`：主要页面存放目录，每个 HTML 文件对应一个独立内容。
- `assets/`：共享样式、脚本或图片文件（如存在）。

## 页面归档说明

- 每个页面均为独立完整的 HTML 文件，可直接在浏览器中打开。
- 页面间可能存在相互链接，但均指向本仓库内的文件。
- 归档内容不涉及任何具体域名、商业推广或诱导性信息。
- 页面标题、描述及内容均以中文为主，部分可能包含其他语言。

## 维护与贡献

本仓库由维护者定期更新。若您希望提交改进或新增页面，请遵循以下原则：
- 新增页面应放入 `pages/` 目录，并确保与现有结构一致。
- 页面内容应保持中立、客观，不包含广告或敏感信息。
- 提交前请检查 HTML 语法正确性及文件编码（建议 UTF-8）。

提交 Pull Request 或 Issue 前，请简要说明修改目的。

## 使用方式

- 直接克隆仓库到本地，在浏览器中打开 HTML 文件即可浏览。
- 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能发布为静态站点。
- 不建议直接引用本仓库中的文件作为外部资源，除非明确授权。
